# 第一版验收清单

## 机器检查

上线前先执行：

```bash
npm run verify
```

通过后，说明代码检查和生产构建没有发现阻断问题。

## 人工验收流程

### 1. 市场行情

1. 打开首页 `/`。
2. 确认默认自选能显示。
3. 搜索常见标的，例如 `AAPL`、`BTCUSD`、`XAUUSD`。
4. 从首页进入阿U智能体。

### 2. 登录和用户中心

1. 打开 `/login`。
2. 注册或登录。
3. 确认页面显示当前账户。
4. 点击“查看历史记录”，确认能进入 `/history`。
5. 点击“退出登录”，确认退出后历史记录不可读。

### 3. 普通问答和行情问答

1. 打开 `/agent`。
2. 问：`黄金现在多少钱？`
3. 确认回答包含价格、时间和数据来源。
4. 问一个普通研究问题，确认能正常回答。

### 4. 固定能力

1. 生成一份个股分析。
2. 生成一份行业研报。
3. 确认回答右上角可以复制、导出 MD、导出 TXT。
4. 打开历史记录，确认报告已保存。

### 5. 文件分析和阿U记忆

1. 登录状态下上传一个可提取文字的 PDF、DOCX、PPTX、TXT 或 MD。
2. 选择 BP 风险分析或路演稿生成。
3. 等待状态出现“已加入阿U记忆”或“文件、分析和阿U记忆已保存”。
4. 到 Supabase Table Editor 查看：
   - `knowledge_documents` 有新增记录。
   - `knowledge_chunks` 有新增记录。
5. 回到阿U，问：`根据我上传过的资料，总结这个项目的核心风险`。
6. 确认回答开头提示“已参考你保存过的资料”。

### 6. 历史记录

1. 打开 `/history`。
2. 查看历史对话、我的报告、文件分析。
3. 测试复制、导出、删除。
4. 删除后刷新页面，确认记录不再出现。

### 7. 权限隔离

1. 用账号 A 生成对话、报告、文件分析。
2. 退出账号 A。
3. 用账号 B 登录。
4. 确认账号 B 看不到账号 A 的历史记录、文件分析和阿U记忆结果。

## 已知边界

- 扫描版 PDF 可能提取不到文字，需要后续 OCR。
- 当前阿U记忆是第一版轻量检索，资料很多后需要升级专业 embedding。
- 当前还没有 Sentry、PostHog、支付和财经资讯页。
- 当前为单机进程内频率保护；扩展为多台服务器时需要迁移到共享限流服务。
- 正式接入付费订单前，需要增加数据库级订单幂等和支付回调验签。

## 稳定性验收

1. 访问 `/api/health`，确认返回 `200` 和 `status: ok`。
2. 快速连续点击两次发送或生成按钮，确认只执行一次并只扣一次点数。
3. 查看 PM2 日志，确认请求具有 `requestId`、`durationMs` 和 `status`。
4. 验证请求失败不会扣点，重新发起新请求后可以正常完成。
